CLASE 1

Instalación - Conceptos Básicos


CONTROL DE VERSIONES

Solucionan tres problemas del pasado:

  • Proyectos difíciles de gestionar y liderar.
  • Riesgos a sobrescribir con mi código al avance formal del equipo.
  • La centralización y poca probabilidad de trabajar remotamente.

ARQUITECTURA DEL ÁRBOL

  • Working Area

    • Es donde editamos nuestro código.

  • Staging Area

    • Es donde seleccionamos los archivos que conformarán el commit.

  • Repository

    • Es el registro de todo el proyecto.

CONFIGURACIÓN

Definimos Nombre, Correo

  • git config --global user.name "Tu nombre"
  • git config --global user.email "Tu correo"

--global afecta a todos los usuarios del sistema.

Activamos la nterfaz de colores

git config --global user.ui true" (default / true)

Listamos nuestra configuración global.

git config --global --list"
git --help comando.

Comando que se utiliza para saber como funciona cada instrucción de git, al igual que da una breve explicacíon de dicha instrucción.

git init

Inicia el tracking de nuestra carpeta

git status

Estado en el que se encuentra nuestro proyecto

git add

agregamos al staging area los archivos seleccionados Agregamos todos los archivos.

git add -A

Agregamos solo el archivo index.html o el nombre del archivo que usemos.

git add index.html
git commit -m "mensaje"

Realizamos el commit con los archivos seleccionados.

git log

Nos dice como está el repositorio a nivel de commits (Cuantos commits tenemos en el repositorio), quien las hizo, cuando las hizo y básicamente todo.

git log > commits.txt

Genera un .txt con los commits en la raíz del proyecto.

git checkout

Con este comando viajamos a travez del tiempo, y podemos ir a cualquier commit.

Si hacemos un cambio a algún proyecto que tienes atrás, crearás dos mundos paralelos

git checkout commit git checkout master Master -> rama principal
git reset

Este comando es súper poderoso y ten cuidado con usarlo, hace lo mismo que git checkout a diferencia que arrasa con todos los commits que tiene enfrente. Tiene tres tipos de ejeción:

  • git reset --soft

    • Es un reset tranquilo, tu Working Area lo deja, no se mete con tú código.

  • git reset --mixed

    • Este reset es raro y muy poco usado, borra tu Staging Area sin tocar tu Working Area

  • git reset --hard

    • Este comando lo que dice es ¡Hey quiero borrar todo! y en efecto eso es lo que hace, inluyendo tu código.
